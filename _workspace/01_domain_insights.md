### 1.2 PDF 추출 관점에서의 형식 특징

- **NEIS 공식 출력본**: 표 기반 PDF, 셀별 텍스트 추출 가능. 단, 학년이 끝날 때마다 페이지가 늘어나며 **항목별 페이지 헤더 패턴**이 반복됨.
- **학교 발급 PDF**: 워터마크/직인이 들어가는 경우가 많고, 한글 폰트가 임베디드 안 된 채 깨지는 사례 다수 → **OCR 폴백 필수**.
- **항목 식별 키워드(파싱 앵커)**: `1. 인적사항`, `2. 학적사항`, `3. 출결상황`, `4. 수상경력`, `5. 자격증 및 인증 취득상황`, `6. 창의적 체험활동상황`, `7. 교과학습발달상황`, `8. 독서활동상황`, `9. 행동특성 및 종합의견`. 학년별로 같은 번호가 반복됨.
- **개인정보 위치**: 인적사항 박스(이름, 주민번호 앞 6자리, 주소, 가족관계), 학적사항(전출입), 보건(학교마다 첨부 여부 다름).

### 1.3 신호 강도 vs 노이즈

| 신호 | 강도 | 이유 |
|------|------|------|
| 자유학기 진로/주제선택 산출물 | ★★★★★ | 학생이 직접 고른 주제 — 흥미 신호 가장 강함 |
| 동아리 3년 일관성 | ★★★★☆ | 누적 = 진정성 |

| 수상 누적 분야 | ★★★☆☆ | 고교 미반영이지만 적성 단서로는 유효 |

| 출결(미인정) | ★★★☆☆ | 건강·생활 이슈, 학종에서 직접 감점 요인 |
| 봉사 시간/유형 | ★★☆☆☆ | 고교는 학교봉사만 인정 — 단순 시간 ≠ 신호 |
| 자격증 | ★★☆☆☆ | 일반계 고교에서는 가산 효과 제한적 |

> **Implication for Product**
> - 파싱 모듈은 **자유학기 활동·동아리·세특·행특·출결**을 1순위로 정확히 추출해야 함.
> - 수상·독서·봉사는 **흥미 가설** 보조 신호로만 쓰고, 직접적인 입시 가산 메시지는 회피해야 함(2024 대입 미반영 항목임을 학생/부모에게 명확히 안내).
> - PDF 파싱 실패 시(OCR 필요, 비정형 양식) **수동 입력 보정 UI**가 반드시 필요.

## 3. 대입 트랙 차별화

### 3.1 트랙 개요와 신호 매트릭스

| 트랙 | 비중(2025~2026 평균) | 핵심 평가 요소 | 약점 신호 | 강점 신호 |
|------|----------------------|----------------|-----------|----------|
| **정시(수능)** | 약 40% (수도권 주요대 40~45%, 서울대 등 일부 ↑) | 수능 표준점수·백분위 | 모의 등급 정체, 약점 영역 누적 | 모의 학평/평가원 안정적 상승 |
| **수시 학생부종합(학종)** | 약 23% (서울권 상위대는 30%+) | 교과 + 세특 + 창체 + 행특 정성 | 세특 빈약, 출결 미인정, 진로 비일관 | 진로 일관성, 발표·탐구 누적, 행특 긍정 키워드 |
| **수시 학생부교과(교과)** | 약 22% (지방·중위권 다수) | 교과 등급(주요교과) | 등급 편차 큼, 비주요교과 낮음 | 전 과목 균질·상위 등급 |
| **수시 논술** | 약 3% (감소 추세) | 논술 + 최저학력기준 | 논리 글쓰기 약점, 최저 미달 가능성 | 논술 적성 + 안정적 수능 영역 1~2개 |
| **수시 실기/특기자** | 약 5~6% | 실기 + 학생부 일부 | 실기 노출 적음 | 실기 누적 경력, 대회/포트폴리오 |
| 기타(농어촌·기회균형 등) | 약 6% | 자격 + 학종/교과 | 자격 미충족 | 해당 자격 충족 |

> 비중은 대학·연도별 편차가 크므로, 서비스 내에서는 **학생이 지망하는 대학군 모집요강 기반**으로 계산해야 한다 (집계치는 가이드 용도).

### 3.2 트랙 추천 알고리즘 단서 (가설)

입력 변수
- `grade_avg_main` — 국·수·영·사/과 주요교과 평균 등급
- `grade_avg_minor` — 비주요교과 평균 등급
- `grade_trend` — 학기별 상승/하락 추세
- `mock_main_avg` — 수능 모의 주요영역 등급
- `mock_trend` — 모의 학평 추세
- `senthuk_density` — 세특 분량·특이키워드 밀도(정성 점수)
- `activity_consistency` — 동아리·진로 일관성 점수
- `attendance_clean` — 미인정 결·지·조 0건 여부
- `track_pref_user` — 학생/학부모가 선호한 트랙

| 조건 | 추천 트랙 | 비고 |
|------|----------|------|
| 내신 1.x~2.x 안정 + 세특 밀도 높음 + 활동 일관 | 학종 우선 + 교과 보조 | 서울권 상위대 가능 |
| 내신 1.x 안정 + 세특 평이 + 모의 평이 | 교과 우선 + 학종 보조 | 학교 지명도·지역균형 활용 |
| 내신 평이 + 모의 1~2등급 안정 | 정시 우선 | N제·실모 비중↑ |
| 내신 등락 큼 + 모의 강함 | 정시 또는 논술 | 논술은 최저 충족 가능 시 |
| 실기 누적 + 내신 평이 | 실기 우선 | 대회·포트폴리오 강화 |

> **Implication for Product**
> - 트랙은 **단일 추천이 아니라 "주력+보조" 듀얼 추천**이어야 한다. 한국 입시는 수시 6장 + 정시 3장으로 사실상 멀티 트랙이다.
> - 추천은 **학년별로 결과가 달라져야** 함 (고1=가설, 고2=확정, 고3=조정).
> - 모의/내신 데이터가 부족한 초중학교 시점 입력에서는 "현재로선 모든 트랙 열려있음 + 신호 강한 트랙 우선" 메시지 유지.

### 4.3 우선순위(Priority) 결정 변수

| 변수 | 영향 |
|------|------|
| **시간 임박도** (마감일까지 D-N) | 가장 큰 가중치 — 학교 시험·원서 접수 등 외부 deadline |
| **학년 단계** | 고1=탐색·기반, 고2=심화, 고3=실전 — 단계마다 우선순위 카테고리 다름 |
| **트랙 적합도** | 추천 트랙에서 가중치 높은 카테고리 우선 |
| **학생 약점 신호** | 약점 영역 보강 TODO를 가시성 있게 끌어올림 |
| **개인화 흥미** | 흥미 신호 일치 시 동기 유발 효과 → 진로/세특 카테고리 우선 |

### 4.4 상태 모델

`backlog → today → in_progress → done | snoozed | skipped`

- `snoozed`는 시기 자체가 안 맞을 때 (예: 고1 5월에 수시 원서 작성 — 자동 snooze).
- `skipped`는 학생이 명시적으로 거부한 경우 — 이유를 받아 추천 알고리즘 보정.

> **Implication for Product**
> - 카테고리 코드를 **데이터 모델 enum**으로 박아두면 향후 통계·필터·시각화 모두 동일 코드로 처리 가능.
> - "오늘의 TODO 5개 (각 카테고리에서 1개씩)" 같은 **밸런스 룰**이 멘탈 케어 측면에서 중요.

## 5. 개인화 신호 식별 (초중학교 생기부 → 고교 코칭)

### 5.1 흥미·관심 분야 신호

| 출처 | 패턴 | 추정 신호 |
|------|------|----------|
| 자유학기 진로 산출물 제목 | "환경", "AI", "역사 인물 탐구" | 해당 주제 흥미 강함 |
| 동아리 누적(초→중) | 같은 분야 2년+ | 일관 흥미 |
| 수상 분야 편향 | 과학탐구/글짓기/체육 등 한쪽 다수 | 적성 단서 |
| 독서 분야 | 과학 50%↑, 인문 50%↑ | 인문/자연 1차 분기 단서 |
| 행특 키워드 | "과학 탐구를 즐겨 함", "친구를 잘 챙김" | 적성 + 인성 |

### 5.2 강·약점 신호

| 항목 | 강점 패턴 | 약점 패턴 |
|------|----------|----------|
| 교과 성취 (중등 A~E) | 주요교과 A 비중↑ | C/D/E 누적, 특정 과목 편차 |
| 세특·발달상황 | "발표력 우수", "근거 들어 토론" | "수업 집중 보완", "과제 미제출" |
| 출결 | 개근/우수 | 미인정 결·지 다수 |
| 봉사 | 정기·꾸준 | 일회성·제로 |
| 행특 | "리더십", "협력", "탐구심", "책임감" | "주의 산만", "감정 기복", "교우관계 보완 필요" |

### 5.3 진로 가설 생성 룰 (예시)

- IF (자유학기 진로 산출물 = "AI/SW/로봇") AND (수학·과학 성취도 ≥ B) AND (동아리 SW/과학) → **이공·SW 계열 가설**.
- IF (독서 인문 50%↑) AND (글짓기/토론 수상) AND (사회·국어 A) → **인문·사회 계열 가설**.
- IF (체육·예술 수상) AND (행특 "끈기·성실") → **예체능 계열 가설**.
- IF (행특에 "리더십"·"발표"·"공감") AND (사회 활동 다수) → **경영·사범·사회 계열 가설**.
- IF 신호 약함/혼재 → **"탐색 모드" — 고1 진로 탐색 TODO 강화**.

> **Implication for Product**
> - 진로 가설은 **확정이 아니라 가설로** 표시해야 한다 ("~~ 계열일 가능성이 보입니다. 다음 활동으로 검증해보세요" 톤).
> - 가설 → 검증 활동(고1 진로 TODO) → 재평가 루프가 핵심 사용자 경험.
> - 신호가 약하거나 부정 신호(낮은 성취, 미인정 결석 등)일 때의 **언어 톤 가이드라인**이 필수 (학생/학부모가 상처받지 않도록).

### 7.2 법·표준

- **개인정보보호법(2024 개정)** — 만 14세 미만 법정대리인 동의, 민감정보 별도 동의.
- **교육관계법령** — 학생 정보의 제3자 제공 제한.
- **GDPR/CCPA**(해외 거주자 사용 가능성 있다면) 추가 검토.

### 7.3 처리 원칙(권고)

1. **로컬 우선**: 가능하면 PDF 파싱은 클라이언트(또는 사용자 기기) 단에서 수행.
2. **서버 저장 시 암호화**: 저장 필요 시 필드별 AES-256, 열람 로그 기록.
3. **자동 폐기**: TODO 생성 후 X일(예: 30일) 후 원본 PDF 자동 삭제.
4. **명시 동의 UX**: "민감정보 항목 추출/저장 동의"를 분리 체크.
5. **탈취·유출 시 통지**: 72시간 내 통지 절차 SOP.

> **Implication for Product**
> - **데이터 흐름도(DFD)**에 민감정보 경로를 별도 색으로 표시하고, 저장 위치·기간을 스펙에 명시.
> - 만 14세 미만 보호자 동의 흐름은 **고1 미만 사용자 시나리오에서 필수**.
> - 학교폭력 조치사항·보건 항목은 **추출하지 않음**을 시스템 정책으로 못 박는다.

## 8. 도메인 리스크·실패 모드

| 리스크 | 발생 시 결과 | 완화 |
|--------|------------|------|
| **PDF 파싱 실패율 높음** (학교마다 양식 변형) | TODO가 빈약하거나 잘못 생성 | OCR 폴백 + 수동 보정 UI 필수 |
| **2028 개편 미반영** | 현 중3에게 잘못된 가이드 | 사이클별 룰셋 분기, 메인 화면에 "적용 입시 사이클" 표시 |
| **대학·전형 변동** | 모집요강 매년 바뀜 | 모집요강은 캐시 + 출처 링크, "최신화 책임은 사용자에게" 면책 명시 |
| **민감정보 유출** | 법적 책임, 신뢰 붕괴 | 7장 처리 원칙 준수 |
| **학생/학부모 갈등 유발 표현** | 서비스 거부 | 톤 가이드라인 — 약점 표현은 행동 제안형으로 ("부족"X, "다음 단계로"O) |
| **부모 입력 vs 학생 입력 충돌** | 누구의 TODO인지 모호 | "사용자 역할" 파라미터, 화면 분기 |
| **사교육 권유로 오해될 표현** | 공교육 외부 부담 가중 비판 | 공식 자원(EBS, 학교 프로그램) 우선 추천, 사교육은 사용자가 명시 요청 시만 |
| **합격 보장 오해** | 클레임 위험 | 모든 추천에 "참고용" 면책 표시 |

> **Implication for Product**
> - **면책·톤 가이드라인**은 spec의 "콘텐츠 정책" 섹션으로 별도 문서화.
> - 추천 결과에는 항상 **신뢰도/근거**(어느 신호로 추천했는지) 노출.

## 9. 입력→출력 처리 관점 데이터 플로우 (요약)

```
[1] PDF 업로드
   └─ 형식 판별 (텍스트 임베디드 vs 스캔)
        ├─ 텍스트형 → pdfminer/pdfplumber 추출
        └─ 스캔형  → OCR (한국어 모델)

[2] 항목 파싱 (앵커 키워드 기반)
   └─ {학년, 학기, 항목코드, 본문} 정규화

[3] 신호 추출
   ├─ 정량: 등급, 출결 카운트, 봉사 시간
   ├─ 정성: 세특·행특 키워드 NLP
   └─ 분야: 자유학기·동아리·독서 주제 분류

[4] 학생 프로파일 생성
   └─ {흥미가설, 강점, 약점, 트랙가설, 컨텍스트(학년/입시사이클)}

[5] 룰 + LLM 결합 TODO 생성
   ├─ 룰 엔진: 학년·학기·트랙별 필수 TODO 시드
   └─ LLM: 개인화 문구·우선순위 조정

[6] TODO 카테고리·우선순위 정렬

[7] 사용자 검토·수정 → 저장
```

> **Implication for Product**
> - 룰 엔진과 LLM의 책임 분담을 스펙에서 명확히 (룰 = 빠뜨리면 안 되는 것, LLM = 개인화/표현).
> - "왜 이 TODO를 추천했는가"의 **explanation** 필드를 데이터 모델에 포함.

## 10. 학교 유형·지역별 변수 (Edge Case)

| 변수 | 영향 | 스펙 처리 |
|------|------|----------|
| 일반계 vs 자사·특목 | 내신 등급 분포·교과 운영 다름 | 학교 유형 입력 필수 |
| 광역 일반고 vs 자율형 공립 | 동아리·진로 프로그램 차이 | 정보 제공 차원에서 표시 |
| 직업계(특성화고) | 입시 전형 자체 다름 | MVP 범위에서 **제외** 또는 별도 트랙 |
| 검정고시 출신 | 생기부 부재 | MVP 범위 **제외**, 별도 안내 |
| 해외 IB·AP 학교 | 양식 자체 다름 | MVP 범위 **제외** |
| 지역(수도권/비수도권) | 모집요강·균형선발 차이 | 학교 소재지 입력 |

> **Implication for Product**
> - **MVP 대상은 "일반계 고교 진학 예정자(현 초·중학생) + 일반계 고교 재학생"**으로 한정 권고.
> - 이외 케이스는 v1 이후 로드맵에 명시.

## 11. 학년별 TODO 시드 예시 (룰 엔진 초기값)

> 이 시드는 **모든 학생에게 공통으로 적용되는 기준선**. 개인 신호로 가중치/표현이 달라진다.

### 고1 — 1학기 (3월 1일 ~ 7월 31일)

```
A1. 1학기 학사일정 캘린더 등록
A2. 수행평가 마감일 정리표 작성
A3. 1차 지필 D-30 학습계획 수립
B6. 3월 학평 응시 + 영역별 등급 기록
C2. 동아리 1개 가입 (자율 + 동아리)
C3. 진로 검사(워크넷·커리어넷) 1회
C5. 학교 봉사 활동 1건 (학교 행사 보조 등)
D1. 관심 학과 5개 1차 리서치
D4. 2학년 선택과목 안내 자료 수집
E1. 미인정 결·지 0 유지 체크
E5. 평일 학습 시간 블록 설정 (주 14시간 기준)
```

### 고2 — 1학기

```
A3. 1학기 중간·기말 D-30 학습계획
B7. 수능 영역별 N제 1세트 시작
C1. 세특용 발표 1건 (관심분야 심화)
C3. 진로 인터뷰 1회 (현직자 또는 대학생)
D3. 트랙 결정 1차 ("주력+보조" 정의)
D4. 3학년 선택과목 1차 결정
E2. 수면 6.5h+ 유지 챌린지
```

### 고3 — 1학기

```
A3. 1학기 중간·기말 — 마지막 내신 산정
B6. 6월 평가원 모의 응시 + 분석
B7. 수능 영역별 약점 마감 캘린더
D2. 지망 대학 모집요강 비교표 작성
D3. 수시 6장 1차 후보 선정
D5. 추천서 요청 대상 교사 확정
E2. 체력·수면 관리 — 8월 이후 유지가 핵심
```

> **Implication for Product**
> - 위 시드를 **JSON 룰 파일**로 가지고 있어야 하며, 각 시드 항목에는 `{id, category, grade, term, mandatory, signal_modifiers}` 구조가 필요.

## 12. 프로덕트 결정에 필요한 핵심 입력값 체크리스트

> product-strategist와 spec-writer가 **반드시 결정해야 하는** 항목들. 이 리스트를 그대로 청사진/스펙에 들고 가서 ✓로 채우면 된다.

### 12.1 사용자/범위
- [ ] **타깃 사용자 세그먼트**: 학생 본인 / 학부모 / 둘 다
- [ ] **타깃 학년 범위**: 초6~중3 (예비 고교생) / 고1~3 / 양쪽
- [ ] **MVP 학교 유형**: 일반계 한정 / 특목·자사 포함 / 직업계 포함
- [ ] **지역 범위**: 전국 / 수도권 한정
- [ ] **언어**: 한국어 only / 영어 병기 여부

### 12.2 입력
- [ ] **PDF 입력 채널**: 업로드 / 카메라 스캔 / 둘 다
- [ ] **수동 보정 UI 범위**: 항목별 텍스트 편집 가능 범위
- [ ] **OCR 사용 여부**: 필요 / 선택 / 사용 안 함
- [ ] **입력 정보 외 보충 입력**: 학년·트랙 선호·관심 학과 등
- [ ] **민감정보 처리 동의 흐름**: 단일 / 분리 / 보호자 동의

### 12.3 출력 — TODO
- [ ] **TODO 단위**: 일/주/월/학기 — 권장 = 학기 단위 + 주 단위 슬라이스
- [ ] **카테고리 체계 채택**: 본 문서 §4 5×다 분류 채택 여부
- [ ] **우선순위 알고리즘**: 단순 시간순 / 가중치 기반 / 학습형
- [ ] **추천 근거(explanation) 노출**: O/X
- [ ] **추천 톤 가이드라인**: 학생용 / 학부모용 분리 여부

### 12.4 트랙·결정 도우미
- [ ] **트랙 추천 방식**: 단일 / 듀얼(주력+보조)
- [ ] **트랙 추천 신뢰도 표기**: 정성 라벨 / 수치 점수
- [ ] **모집요강 데이터 소스**: 공식 API / 수동 수집 / 사용자 입력
- [ ] **2028 개편 룰셋 분기**: 입학년도 자동 계산 / 사용자 선택

### 12.5 데이터·보안
- [ ] **저장 정책**: 클라이언트 only / 서버 암호화 저장 / 둘 다 옵션
- [ ] **원본 PDF 보존 기간**: 미보관 / 7일 / 30일 / 사용자 지정
- [ ] **민감정보 폐기 룰**: 추출 즉시 / 세션 종료 시 / N일 후
- [ ] **로그·감사 정책**: 열람 로그 보관 기간

### 12.6 콘텐츠 정책
- [ ] **면책 문구 위치**: 결과 화면 / TODO 항목별 / 모두
- [ ] **사교육 추천 정책**: 비추천 / 사용자 요청 시만 / 자유
- [ ] **수상·독서 안내**: 흥미 신호로만 / 명시적 미반영 안내 포함
- [ ] **학교폭력·보건 항목**: 추출 안 함 (권장) / 사용자 동의 시만

### 12.7 메트릭(KPI)
- [ ] **핵심 지표**: TODO 완료율 / 트랙 적합도 변화 / 사용 지속률
- [ ] **학습 지표**: 학년 진학 후 실제 활동 매칭률 (장기)
- [ ] **품질 지표**: PDF 파싱 성공률, 추천 만족도

### 12.8 비기능
- [ ] **응답 시간**: 업로드→TODO 생성 ≤ N초 (권장 30초 이내)
- [ ] **오프라인 지원**: 필요 / 선택 / 미지원
- [ ] **다중 디바이스 동기화**: 필요 / 미지원
