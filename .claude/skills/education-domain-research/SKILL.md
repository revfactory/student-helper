# Education Domain Research

학생 맞춤 TODO 생성 서비스의 도메인 기반을 다지는 스킬. 한국 교육 시스템과 대입 제도, 생활기록부 구조에 대한 정확한 이해를 바탕으로 프로덕트가 다뤄야 할 도메인 신호를 정리한다.

## 워크플로우

### 1. 입력 확인
- 사용자 원본 요청 또는 오케스트레이터로부터 받은 컨텍스트
- 기존 `_workspace/01_domain_insights.md` 존재 여부 확인 → 존재 시 갱신 모드, 없으면 신규 작성

### 2. 핵심 분석 영역

다음 5개 영역을 모두 다룬다:

#### 2-1. 생활기록부(NEIS) 구조 및 신호 매핑
초등/중등 생기부의 주요 항목별로 의미와 TODO 단서를 정리한다.

| 항목 | 초등 | 중등 | 의미/신호 | TODO 단서 |
|------|------|------|----------|----------|
| 인적사항 | ○ | ○ | 기본 식별 | 개인화 키 |
| 학적사항 | ○ | ○ | 전학·이탈 패턴 | 적응 이슈 신호 |
| 출결상황 | ○ | ○ | 성실도 | 자기관리 TODO |
| 수상경력 | × | ○ | 강점 영역 | 강점 심화 TODO |
| 자격증 | × | ○ | 외적 인증 | 자격 로드맵 |
| 창의적 체험활동 | ○ | ○ | 자율/동아리/봉사/진로 | 비교과 TODO |
| 자유학기/학년 활동 | × | ○ | 진로탐색 | 진로 심화 TODO |
| 교과학습발달상황 | ○ | ○ | 학업 강약점 | 학습 TODO |
| 행동특성 및 종합의견 | ○ | ○ | 인성·성향 | 멘탈/관계 TODO |

각 항목에서 어떤 신호가 어떤 TODO로 이어지는지 구체 예시 1~2개씩 작성.

#### 2-2. 고교 학년별 핵심 과업

**고1 (탐색기 / 3월~다음 해 2월)**
- 학습: 기초 학력 진단, 내신 첫 시험, 학습 습관 정착
- 진로: 적성 탐색, 1지망 학과 가설, 첫 진로 활동
- 비교과: 동아리 가입, 자율활동, 봉사 시작
- 입시 준비: 수시/정시 트랙 1차 가설
- 자기관리: 출결, 멘탈, 학교 적응

**고2 (심화기)**
- 학습: 내신 + 모의고사 균형, 약점 보강
- 진로: 학과 확정 → 활동 심화 (탐구, 독서, 프로젝트)
- 비교과: 리더십, 심화 동아리, 전공 연계 활동
- 입시: 트랙 확정 (학종/정시/논술 등)

**고3 (실전기)**
- 학습: 수능 대비 / 내신 마무리 / 면접·논술 준비
- 진로: 자기소개서, 추천서, 학과별 전형 분석
- 입시: 수시 6장 전략, 정시 지원 시뮬레이션
- 자기관리: 컨디션 관리, 멘탈

각 학년별로 월별 또는 분기별 마일스톤 포함.

#### 2-3. 입시 트랙별 차별화

- **정시 (수능 위주)**: 수능 점수 극대화, 모의고사 분석, 영역별 학습 전략
- **수시 학생부종합전형 (학종)**: 학생부 활동 충실도, 자소서, 면접
- **수시 학생부교과전형 (교과)**: 내신 등급, 출결, 비교과는 보조
- **수시 논술전형**: 논술 학원/연습, 논술 유형별 대비
- **수시 실기전형 (예체능)**: 실기 능력, 포트폴리오, 입시 대회

각 트랙별로 "어떤 학생에게 적합한가" + "TODO 차별 포인트" 정리.

#### 2-4. TODO 카테고리 체계

표준 카테고리 제안 (1차/2차):
- **학습**: 내신, 수능, 자기주도학습
- **진로**: 탐색, 심화, 결정
- **활동**: 동아리, 자율, 봉사, 진로활동
- **시험**: 내신, 모의고사, 수능, 자격증
- **자기관리**: 멘탈, 건강, 시간관리
- **입시**: 트랙 결정, 자소서, 면접, 지원

각 카테고리별 우선순위 기준 (시급성, 영향도, 학년).

#### 2-5. 개인화 변수

생기부에서 추출 가능한 개인화 변수:
- 학업 성향 (이과형/문과형, 깊이/속도형 등)
- 관심 분야 (수상·동아리·진로활동 빈도 분석)
- 활동 적극성 (창체활동 풍부도)
- 약점 영역 (과목별 점수, 출결, 행특)
- 사회성/리더십 신호 (행특, 동아리 직책)

### 3. 도메인 리스크/주의사항

- 입시 제도 변경 빈도 (매년 변동) → 데이터 갱신 메커니즘 필요
- 학교별 차이 (수상 풍토, 동아리 종류 등) → 일반화 한계 인식
- 생기부 비공개 항목 (예전 기록부) → 입력 데이터 한계
- 민감정보 처리 (이름, 학교, 가족 정보 등)

### 4. 산출물 작성

`_workspace/01_domain_insights.md`에 references/template.md 형식으로 작성한다.

본문은 ~600줄 이내로 유지하고, 더 깊은 자료(예: 트랙별 통계, 입시 변경 이력)는 references/ 하위로 분리.

## 작성 원칙

- **사실 기반**: 추정이 아닌 공식 제도(2015/2022 개정 교육과정, 대입공정성 강화 방안 등)에 근거. 불확실하면 명시.
- **현장 감각**: 실제 학생/학부모/교사가 마주하는 고민과 실수 반영.
- **실행 가능성**: 제시 활동은 일반 학생이 수행 가능한 수준으로.
- **구체 사례**: 추상적 설명보다 "예: ○○ 동아리 활동을 한 학생은 □□ 활동으로 연결" 같은 구체 사례 1~2개 동반.

## 후속 작업 모드

- **신규**: 위 워크플로우 전체 수행
- **갱신**: 사용자가 지정한 섹션만 보강. 다른 섹션 보존.
- **사례 추가**: 신호 매핑 테이블에 새 사례 1~2줄 추가.
